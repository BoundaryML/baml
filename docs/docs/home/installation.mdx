title: Installation
---

<Steps>
  <Step title="Install BAML VSCode Extension">
    [https://marketplace.visualstudio.com/items?itemName=boundary.BAML](https://marketplace.visualstudio.com/items?itemName=boundary.BAML)

    If you are using python, [enable typechecking in VSCode's](https://code.visualstudio.com/docs/python/settings-reference#_python-language-server-settings) `settings.json`:
    ```
    "python.analysis.typecheckingMode": "basic"
    ```
  </Step>
  <Step title="Install BAML">
    <CodeGroup>
        ```bash Python
        pip install baml-py
        ```

        ```bash Typescript
        npm install @boundaryml/baml
        ```
    </CodeGroup>
  </Step>
  <Step title="Add your first BAML files">
      <CodeGroup>
        ```bash Python
        # Should be installed via pip install baml-py
        baml-cli init
        ```

        ```bash Typescript (npx)
        npx baml-cli init
        ```

        ```bash Typescript (pnpx)
        pnpx baml-cli init
        ```
    </CodeGroup>
  </Step>
  <Step title="Learn BAML">
    - [PromptFiddle](https://promptfiddle.com): Interactive examples to learn BAML. (recommended)
    - [BAML Tutorials](docs/guides): Advanced guides on using BAML.
    - [BAML Syntax](/v3/syntax): Documentation for BAML syntax.
    - [BAML Starters for NextJS and FastAPI](https://github.com/BoundaryML/baml-examples/tree/main)
  </Step>
</Steps>

## Ensure BAML CLI can generate your Python / TS client

    <Tabs>
    <Tab title="VSCode">
    Save a `.baml` file using VSCode, and you should see a successful generation message pop up!
    </Tab>
    <Tab title="terminal">
    ```bash
    # Run from any dir/subdir in your project
    # Requires you have a baml_src directory with a main.baml file
    > baml build
    ```
    </Tab>
    </Tabs>
